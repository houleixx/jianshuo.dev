VoiceDrop 为 Agent 而生

你开口说，VoiceDrop 把它变成你自己云端的结构化文章 —— 一个 token，
Claude Code、Codex 这样的 agent 就能直接读、直接写。

为什么说它是 agent-native

VoiceDrop 不是一个把你的数据锁在自己后台的 app。每条录音和挖出来的文章，都存在 你自己的空间里，背后是一套极简的、token 鉴权的开放 API。也就是说，你的 voice memo 从录下来的那一刻起，就已经是 agent 可以读写的结构化数据。

  • 开放 Files API：列文件、下载录音 / 文章 / 字幕，全部一个 curl 搞定。
  • 文章是结构化 JSON（标题 + 正文 + 转写 + SRT），不是一坨 HTML。
  • 一个 CLAUDE.md 的反馈回路：让 agent 蒸馏你的文风写回去，下一篇就更像你。
  • 数据在你自己的目录里，token 在你手上 —— 无缝融入你的 agent 工作流。
在 Claude Code / Codex 里用 · How to use
1

拿到你的访问令牌

打开 VoiceDrop App → 底部「设置」→「账户」→ 点「复制访问令牌」。令牌形如 anon_xxxx…。

2

在 Claude Code / Codex 里设好它

TOKEN=anon_xxxxxxxx...   # 粘贴你刚复制的令牌
BASE=https://jianshuo.dev/files/api
3

列出 / 读取你的录音和文章

# 列出你空间里的所有文件
curl -s -H "Authorization: Bearer $TOKEN" $BASE/list

# 下载某篇文章的结构化 JSON
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/download/users/<你的id>/articles/VoiceDrop-....json"
4

或者直接用大白话指挥 agent

装上 voicedrop skill 后，在 Claude Code 里直接说：

「voicedrop list」—— 列出我所有的录音和文章
「voicedrop distill」—— 从我最近几篇文章蒸馏文风，写回设置里的「文风」

agent 会用上面的 API 帮你把活干完，蒸馏出的文风规则会直接写进你的 CLAUDE.md，下次挖文章自动带上。

这条回路有多顺

说一段话 → VoiceDrop 自动转写、挖成文章 → 你让 agent 读这些文章、蒸馏出你的文风 → 写回 CLAUDE.md → 下一篇文章更像你。整个过程里，agent 始终在用你自己的数据、 你自己的 token，没有第二个后台、没有锁定。这就是 VoiceDrop 想做的事：让口述无缝接进 agent 生态。
