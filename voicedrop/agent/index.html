Agent 使用指南

在 Claude Code / Codex
里用 VoiceDrop

你的每一段录音，都已经是 agent 可以读写的结构化数据。
以下是全部可用的 slash 命令和 skill。

第零步：拿到访问令牌
1

在 VoiceDrop App 里复制令牌

底部「设置」→「账户」→ 点「复制访问令牌」。令牌形如 anon_xxxx…。

2

在 Claude Code / Codex 里设好

TOKEN=anon_xxxxxxxx...   # 粘贴你刚复制的令牌

令牌只在本次 shell 会话有效。也可以写进 ~/.zshrc 的 export VOICEDROP_TOKEN=… 里永久保存。

可用 Skill 一览
/wjs-voicedrop

VoiceDrop 配套 skill，两个模式：list（列出你的录音与文章）和 distill（从你的文章蒸馏文风，上传到 CLAUDE.md）。

触发词 / 说什么都行：

/wjs-voicedrop voicedrop list 列出 VoiceDrop 文件 voicedrop distill 蒸馏 VoiceDrop 文风

模式一：list — 列文件

说「voicedrop list」，agent 会列出你在 VoiceDrop 空间里的所有内容并分类：

  • 待处理录音（还没变成文章的 .m4a）
  • 已成文文章（articles/*.json，每篇含标题 + 正文 + 转写 + SRT）
  • 无语音标记（articles/*.empty，原始录音里没说话）
  • 文风设置（你上传的 CLAUDE.md）
「voicedrop list」
「帮我看看 VoiceDrop 里还有哪些录音没变成文章」

模式二：distill — 蒸馏文风

说「voicedrop distill」，agent 会从你最近的几篇文章里提炼 15–20 条可执行的写作规则，预览后确认，上传到你的 CLAUDE.md——下次挖文章时服务器自动带上。

「voicedrop distill」
「帮我从 VoiceDrop 文章里蒸馏文风，更新到设置里」

上传成功后，在 VoiceDrop App 的「设置→文风」里也能看到，两边是同一份数据。

/wjs-distilling-style

通用文风蒸馏 skill。给任意几篇文章，提炼出这位作者精确可执行的写作指纹——不只是 VoiceDrop 用户，也可以用来学任何作者的笔法。

触发词：

/wjs-distilling-style 蒸馏文风 提炼 XX 的风格 把这篇改成 XX 的味儿 mimic this writer

两个模式

  • 蒸馏：给 3–6 篇样本文章，输出一张 9 轴 Style Card（句子节奏、段落密度、词汇、语气、论证结构、比喻、情绪、开头/收尾、雷区），存到 ~/code/style-cards/<author-slug>/style-card.md。
  • 改写：给一篇目标文章 + 已有 Style Card，改写成对应作者的笔调。改写后有独立「裁判」盲测，不让 agent 自己给自己打分。
「把这三篇文章的文风蒸馏出来，存成 Style Card」
「用王建硕的文风把这篇稿子改一遍」

和 /wjs-voicedrop distill 的区别：这里适合精细化全量 9 轴蒸馏；voicedrop distill 是快速模式，只提炼差异规则补充到 VoiceDrop 的 CLAUDE.md。

直接用 API（不走 skill）

Files API 速查

所有 skill 底层都在调这套 API，你也可以在 agent prompt 里直接用：

BASE=https://jianshuo.dev/files/api

# 列出所有文件
curl -s -H "Authorization: Bearer $TOKEN" $BASE/list

# 下载某篇文章 JSON
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/download/users/<你的id>/articles/VoiceDrop-xxxx.json"

# 下载文风设置
curl -s -H "Authorization: Bearer $TOKEN" \
  "$BASE/download/CLAUDE.md"

# 上传更新文风设置
curl -s -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: text/markdown; charset=utf-8" \
  --data-binary @/tmp/style.md \
  "$BASE/upload/CLAUDE.md"

文章 JSON 格式：{ title, body, transcript, srt }。

整条回路长这样

说一段话 → VoiceDrop 自动转写、挖成文章 → /wjs-voicedrop list 读出来 → /wjs-voicedrop distill 或 /wjs-distilling-style 蒸馏文风 → 写回 CLAUDE.md → 下一篇更像你。

整个过程，agent 始终在用你自己的数据、你自己的 token，没有第二个后台，没有锁定。这就是 VoiceDrop 想做的事：让口述无缝接进 agent 生态。
