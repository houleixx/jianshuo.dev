每次授权弹窗距上一次的间隔（天）

每根柱子是一次「x.x.xxx 想访问其他 App 的数据」弹窗——Claude Code 每天自动更新一个版本，macOS 就把它当成一个全新程序，重新要一次授权。

1

可执行文件按版本号命名

Claude Code 的命令行主程序放在 ~/.local/share/claude/versions/2.1.199——文件名就是版本号，弹窗标题也就成了「2.1.199」。

2

macOS 按路径记录授权

对没有 App 包的裸可执行文件，隐私授权（TCC）按绝对路径记账。路径一变，之前的「允许」全部作废。

3

每天自动更新 = 每天新程序

Claude Code 几乎每天自动升级一个版本，每个新版本都是一条新路径——于是弹窗天天回来。这是官方已知 bug。

✓ 已做的修复

  • 在 ~/.claude/settings.json 写入 "DISABLE_AUTOUPDATER": "1"，关掉每日自动更新（这是 GitHub issue 里确认的唯一根治办法）。
  • 下次弹窗（针对当前 2.1.199）点一次「允许」，之后就安静了。
  • 以后想升级时手动跑 claude update；每次手动升级后会再问一次，点允许即可。
